0. A solid code base

Solid base = valid code = robust code.

Code is invisible to most users, but not to assistive technology.

  • Code

  1. Document Type

    <!DOCTYPE html>

    The doctype is your website's building code. Use a strict doctype and validate your pages.

  2. A Page Title

    <title>A Unique Page Title</title>

    A unique title clearly tells the user where he/she is.

  3. Character Encoding

    <meta charset="UTF-8">

    Ever see funny little symbols instead of letters and punctuation? It was likely having an encoding problem.

1. Text: accessible by nature

To ensure text is even more accessible to all users:

  • Content
  • Code

  1. Declare the language

    The browsers and assistive technologies such as screen readers need to know what language the content is written in, and it needs to made aware of language changes.

    <html lang="en">

    <q lang="fr">Bonjour!</q>

  2. Code meaning into content

    By giving it meaning document structure: headings, lists, paragraphs, quotations, emphasis, tables, etc.

  3. Natural linear order

    A user should be able to read the content from top to bottom (depending the language, of course) with styles turned off.

  4. Appropriate reading level

    Keep it as simple as possible.

2. Links & Navigation

The stepping stones of the internet. Well-planned, consistent navigation benefits all users.

  • Content
  • Code

  1. Keep navigation consistent

    All users love consistency. Consistency has to persist in the code as well. As for a screen reader the heading hierarchy is a form of navigation.

  2. Skip to Content links

    Special links (often hidden) that provide a direct route to the main content are and other landmarks.

    HTML5 has a robust set of new landmark tags, special tags for way finding.

  3. Use unique meaningful words for links

    Clear function: avoid generic click here statements.

3. Images & Charts

A text-based alternative for all meaningful visual content is required.

  • Content
  • Code

  1. Short descriptions

    <img src="image-file.jpg" alt="Description." >

    • alt text(hidden to visual user)
    • caption (visible to all users)
    • content (visible to everyone)

  2. Long descriptions

    May need longer descriptions than is appropriate for the alt attribute.

    • Link to a supplementary description.
    • Goal: maintain the meaning of the document when the image is removed.

4. Tables

Use the rich set of semantic table mark-up that is available.

  • Content
  • Code

  1. Keep tables simple

    And use them for tabular data, only. CSS is for layout.

  2. Caption your tables

    <caption>Introduce the table here.</caption>

    Table captions are associated directly with the table and give a visually impaired user a brief description of the data and the layout.

  3. Scope out direction

    <th scope="row">Describe the table here.</th>

    Use the scope attribute to give the row or column content direction.

5. Colour & Layout

Accessible colour and layout choices need to be agreed upon at the beginning of project.

  • Design
  • Code
  • Content

  1. Choose colours wisely

    • Don't rely on colour alone for meaning

      Use formatting or icons to also communicate the meaning.

    • Choose your standard (AA or AAA)

      (4.5:1 or 7:1). Make it your practice to test for contrast.

    • Build awareness around colour confusion

      The WCAG Samurai recommend trying the Brewer palette.

  2. Use CSS for Layout

    Using CSS & HTML together properly gives you the biggest bang in making your content accessible.

6. Forms: complex by nature

Keep things simple. Don't ask for data you don't need.

  1. Label all form controls

    Associate a label to every form control.

  2. Keep form elements organized

    Forms have a rich set of tags: legend, fieldset, optgroup. Grouping like controls together creates organization and meaning.

Note: Inaccessible forms & complex data tables have been the basis for lawsuits.

7. Video & Audio

  1. No auto play.

  2. Like images, video & audio need text alternatives to be made accessible. This can be achieved through:

    • Accessible Transcripts {demo video}
    • Captions (closed or open)
    • Audio description (aka video description)
    • Subtitles for language accessibility

Interactive web components such games and learning exercises are more complex.

  2. Graceful degradation.
